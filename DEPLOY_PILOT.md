# Deploy for the Driver Pilot — Atlas + Render (single origin)

Goal: a **24/7 public HTTPS URL** real drivers can use from their phones.
One Render web service serves **both** the React app and the API from the same
origin (verified locally by `backend/scripts/smokeProd.js` — run it after any
build-related change).

Total time: ~45 min. Cost: Render **starter $7/mo** + Atlas **M0 free**.
Payments stay OFF (no Stripe keys) — this is the no-money pilot boundary.

---

## Step 1 — MongoDB Atlas (~10 min, your account)

1. https://cloud.mongodb.com → create a free **M0** cluster (region: **Oregon/us-west-2** to match Render's `oregon`).
2. **Database Access** → Add user → username `fcpilot`, autogenerate a strong password → role **Read and write to any database**.
3. **Network Access** → Add IP → **0.0.0.0/0** ("allow from anywhere" — Render's IPs rotate; auth + TLS still protect you).
4. **Connect → Drivers** → copy the connection string and set the db name:
   ```
   mongodb+srv://fcpilot:<password>@<cluster>.mongodb.net/freightconnect?retryWrites=true&w=majority
   ```
   This is your **MONGO_URI**. Keep it out of git.

## Step 2 — Render (~15 min, your account)

1. https://render.com → **New → Blueprint** → connect the GitHub repo
   (`sergheiganenco/FreightConnect`). Render reads `render.yaml` automatically.
2. When prompted for the `sync: false` env vars, set **only**:
   - `MONGO_URI` = the Atlas string from Step 1
   - `ALLOWED_ORIGINS` = your service URL, e.g. `https://freightconnect.onrender.com`
     (you'll see the exact URL on the service page; comma-add more origins later if needed)
   - Leave `STRIPE_*`, `EMAIL_*`, `ORS_API_KEY`, `FMCSA_API_KEY` **empty** — the app
     degrades gracefully (payments off, email off, FMCSA mock). `JWT_SECRET` and
     `TRACKING_TOKEN_SECRET` are auto-generated by the blueprint.
3. Deploy. First build takes ~5–10 min (frontend build + backend install).
4. Verify: `https://<your-url>/api/health` → `{"status":"ok","db":"connected",...}`
   and `https://<your-url>/` → the FreightConnect app loads.

## Step 3 — Seed the admin (~2 min, from your machine)

```powershell
cd backend
$env:MONGO_URI = "<the Atlas string>"
node scripts/createAdmin.js admin@yourco.com "AStrongPassword123!" "Your Name"
```

Then log in at `https://<your-url>/login` as the admin and create the pilot
accounts via **Dashboard → Users → Create User**:
- 1 shipper (you or a friendly shipper)
- 1–2 carriers (the real drivers)

> No SMTP in the pilot ⇒ no email verification / password reset. Hand accounts
> out directly and pick good passwords.

## Step 4 — Driver phone setup (~5 min per driver)

The driver does NOT need an app install for the pilot:
1. **Web app on the phone**: open `https://<your-url>` in Safari/Chrome → log in
   → (optional) "Add to Home Screen".
2. **GPS consent**: Profile → GPS Tracking Consent → grant. (Tracking is blocked
   server-side until this is done.)
3. **Background GPS with the screen off** (recommended for real hauls): install
   the free **OwnTracks** or **Traccar Client** app. When a load is accepted, the
   Load Details screen exposes a per-load tracker URL (`/api/tracking/ingest/...`)
   — follow **TRACKING_FREE_IOS.md** for the 2-minute setup.
   - For quick tests, the web app's in-browser tracking works while the screen is on.

## Step 5 — Dry run before inviting anyone (~15 min)

Golden path, once, yourself (two browsers or phone+laptop):
1. Shipper posts a load (Post Load wizard).
2. Carrier bids → shipper accepts (or carrier accepts directly).
3. Carrier starts trip → truck appears on the shipper's Load Details map / Fleet Live Map.
4. Drive/simulate movement → geofence check-in at the destination → detention
   documents itself if dwell exceeds free time.
5. Mark delivered. File a test dispute with a photo; resolve it as admin.
6. `PILOT_GUIDE.md` has the fuller script + what to tell participants.

## What to put in the pilot invite

- The URL + their login.
- "Beta: **no payments move through the app** — settle off-platform as usual."
- "The app records GPS only while you're on an active load, for ETA + detention
  proof; you can revoke consent in Profile at any time."
- Where to report issues (your phone/email/group chat).

## Troubleshooting

| Symptom | Fix |
|---|---|
| `/api/health` says `"db":"disconnected"` | MONGO_URI wrong or Atlas network access not 0.0.0.0/0 |
| App loads but login fails / CORS or socket errors | `ALLOWED_ORIGINS` must exactly match `https://<your-url>` (no trailing slash); redeploy after changing |
| White page | Check the Render build log — frontend build must have succeeded; then `node scripts/smokeProd.js` locally to reproduce |
| Driver's GPS not appearing | Consent not granted, load not in-transit, or wrong driver logged in |
| Free-tier sleep (if you chose free instead of starter) | First request after idle takes ~50s; starter plan doesn't sleep |

## When the pilot outgrows this

- Real background GPS in a branded app → EAS build of `mobile/` (ask me).
- Email flows → set `EMAIL_USER`/`EMAIL_PASS`.
- Payments → `STRIPE_GO_LIVE.md` (staged; collection flag stays off until field-tested).
